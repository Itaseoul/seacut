# 벤치 체크리스트 — 보드 5기 (인쇄용)

> **한 줄 요약**: 보드 도착 → 서버(LAN) 기동 → 유닛별 5게이트(G0 플래시 → G1 GPS전원 → G2 풀사이클 → G3 버퍼플러시 → G4 HTTPS) → 공통 2항(슬립 전류·조립 이행). **전 게이트 통과 전엔 조립·방류 없음.** 근거: [firmware/README](../firmware/README.md) 벤치 재검증 체크리스트 · [조립_가이드](조립_가이드.md).

## 0. 사전 준비 (보드 도착 전에 완료)

| ✅ | 항목 |
|---|---|
| ☐ | Arduino IDE + ESP32 core 3.0.x, 보드 "ESP32 Dev Module" |
| ☐ | 라이브러리: **lewisxhe/TinyGSM-fork**(★스톡 TinyGSM 제거), TinyGPSPlus, ArduinoHttpClient |
| ☐ | 수신 서버 v1.1 확인: `python ingest_server.py 8770 0.0.0.0` (LAN 바인딩) — 기동 로그에 `FF-ID schema v1.1` |
| ☐ | 이 PC의 LAN IP 확인(`ipconfig`) → 펌웨어 `SERVER_HOST`에 기입, 방화벽 8770 인바운드 허용 |
| ☐ | (서버 드라이런) curl로 v1.1 ping POST → `ok:true`, 같은 seq 재전송 → `dup:true`, `/api/live.geojson`에 궤적 |
| ☐ | SIM 5장(Soracom/1NCE 또는 KT·SKT 데이터심 — ★LG U+ 불가), 보호 18650 5개(연속 3A+) 충전 |
| ☐ | 유닛 ID 라벨 5장 인쇄: `ff-kr-bs-u0001` ~ `u0005` |

## 1. 유닛별 게이트 (보드마다 반복)

- **G0 플래시**: `DEVICE_ID`를 유닛 고유값으로 바꿔 업로드(`BENCH_HTTP 1`, `SLEEP_MINUTES 5`). ★18650 장착 후 진행(USB 단독 급전 금지 — 2A 피크 브라운아웃).
- **G1 GPS 전원 관문**: 창가/야외에서 첫 웨이크 로그 확인. fix 성공 또는 실패라도 `NMEA chars>0`이면 통과. **`chars=0`이면 중단** → L76K가 모뎀 전원에 의존 → [firmware/README](../firmware/README.md) 체크리스트 1번 조치(modemPowerOn을 getFix 앞으로) 후 전 유닛 재플래시.
- **G2 풀사이클**: fix → POST 200 → 딥슬립 → 웨이크, 3주기 연속. 서버에서 `seq`가 1씩 증가, `ts_fix`·`sample_interval_s`·`batt` 수신 확인.
- **G3 버퍼 플러시**: LTE 안테나 분리 → 2~3주기(로그에 "버퍼 보존" 확인) → 안테나 재결합 → 다음 웨이크에 **밀린 레코드가 오래된 순으로 몰아 전송**되고 지도 궤적이 ts_fix 순으로 조립되는지.
- **G4 HTTPS 빌드**(대표 1기만): `BENCH_HTTP 0`(A76XXSSL)로 컴파일·업로드 → openc로 POST 성공. 확인 후 벤치용은 다시 `BENCH_HTTP 1`.

|---|---|---|---|---|---|
