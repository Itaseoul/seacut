# DIY 다양성 프레임 — 작은 고정 커널 + 넓은 자유 껍질

> **한 줄 요약**: Friendly Floaty는 Precious Plastic·RepRap·Arduino처럼 **소수의 강한 보편 규칙(커널) + 넓은 현지 적응(껍질)** 으로 확산한다. 커널 4개(데이터 스키마·회수-우선·부력체 성능·정직 라벨)만 지키면 나머지 — 소재·부력재·컴퓨트·밴드·폼팩터·펌웨어·회수법·현지화 — 는 전부 열려 있다. **다양성이 커널을 깨지 않는 한.** 이 문서는 규칙을 새로 만들지 않고, 이미 합의된 4개 커널을 *왜 고정인지*·*최소 계약*으로 성문화하고, 자유의 경계와 커뮤니티 메커니즘을 못 박는다. 각 규격의 정본은 원문서에 있다(링크).

---

## 0. 원리 — 다양성은 확산 엔진이고, 커널은 그 경계다

오픈 하드웨어는 **단서(caveat)의 부재가 아니라, 작은 불변 커널 + 넓은 자유 껍질**로 퍼진다. 성공한 무브먼트는 전부 이 구조다.

| 무브먼트 | 고정 커널 (작음) | 자유 껍질 (넓음) | 확산이 작동한 이유 |
|---|---|---|---|
| **RepRap** | G-code · STL · 자기복제 가능성 | 프레임·재료·모터·전자·바디 무한 변종 | 파일 포맷이 고정이라 **모든 파생 프린터가 같은 모델을 출력**. 껍질 폭발이 커널 가치를 키움 |
| **Arduino** | 핀 헤더 배치 · IDE/코어 API | 보드·클론·실드·MCU 무한 변종 | 스케치가 **어느 보드에서나 돎**. API 고정이 생태계를 열되 파편화를 막음 |
| **Precious Plastic** | 플라스틱 종류 규격 · 기계 공차 · **안전 가이드** | 기계 바디·현지 조달·제품 디자인 자유 | "무한정 다양 = 불신뢰"를 **스펙과 상표정책으로 경계** 지어 품질을 유지하며 복제 |
| **OpenMetBuoy** | 데이터/메시지 포맷 | 하우징·전개 방식 | 부이가 달라도 **관측이 합쳐짐** |
| **Friendly Floaty** | **K1 스키마 · K2 회수-우선 · K3 Verified Float · K4 정직라벨** | 하우징·부력재·컴퓨트·밴드·폼팩터·펌웨어·회수법·현지화 | 스키마가 데이터를 합치고, 성능테스트가 신뢰를 지키고, 회수-우선이 환경 정당성을 지킴 |

**이 구조가 확산을 가속하는 세 메커니즘**:

1. **진입 장벽을 껍질로 낮춘다.** 커널만 지키면 되니, 빌더는 자기 나라의 폐부표·현지 SIM·재사용 폰으로 시작한다(§3, Tier0 USD 5~35). "인증 완제품 배급"이 아니라 "테스트를 통과시켜라"이므로 조달이 다른 어느 지역도 배제되지 않는다.
2. **파편화를 커널로 막는다.** 껍질이 폭발해도 데이터는 한 테이블에, 유닛은 전부 회수되고, 뜨는 것만 방류된다. 다양성이 늘수록 공통분모가 데이터뿐이라 **K1 스키마의 가치는 오히려 커진다** — RepRap에서 프린터가 늘수록 G-code 가치가 커진 것과 같다.
3. **감시가 아니라 게이트로 강제한다.** 우리는 전 세계 빌더를 감시할 수 없다. 그래서 커널 위반은 "틀린 것"이 아니라 **"공식 Friendly Floaty가 아니고 공유 데이터셋에 안 들어가는 것"** 으로 처리한다(사회적·기계적 게이트, 물리적 강제 아님).

> 이 문서의 커널 조항 상당수는 원문서를 **더 조이는 강화 패치**를 담고 있다(§5의 가드레일). 강화가 랜딩하는 정본 파일을 각 조항에 표기했다.

## 1. 고정 커널 4개 — 최소 계약 (이게 변하면 무브먼트가 깨진다)

> **연쇄 논리**: K3(뜨고 안테나 산다) → K2(그래서 신호로 회수한다) → K1(회수·검증된 유닛만 데이터가 된다) → K4(그 데이터를 정직하게 라벨한다). 하나라도 빠지면 게이트에서 걸린다: `데이터셋_포함 = unit_id ∧ verified_build(증거有) ∧ provenance(인증 operator) ∧ recover_status(교차검증)`.

### K1 · 데이터 스키마 (poolable data) — 정본 [data/schema.md](../data/schema.md)

**왜 절대 커널인가**: 껍질이 아무리 다양해져도 — 하우징이 병이든 오리든, 컴퓨트가 폰이든 LILYGO든, 밴드가 어디든 — **모든 유닛이 같은 스키마로 POST하면 관측치가 한 테이블에 쌓인다.** 이게 깨지면 100개 마을이 100개의 서로 못 합치는 CSV를 만들고, "글로벌 하천→해양 표류 데이터셋"이라는 존재 이유가 사라진다. 다양성을 허용하는 유일한 방법이 곧 스키마 고정이다.

**최소 계약**:
- (a) `schema_version`을 붙여 POST한다. 버전 범프 시 **하위호환/필드매핑 매트릭스**를 스키마에 상설하고, 혼합버전 집계 규칙을 명문화한다.
- (b) `lat·lon` 필수. 페이로드에 **`crs:"EPSG:4326"` 고정**(다른 값이면 거부). lat/lon **최소 소수 5자리**(≈1m) 요구 — 저정밀 절단이 미세 궤적을 파괴하지 못하게. `batt` 단위는 **volts 고정**.
- (c) **시간축 이원화**: `ts_fix`(디바이스 GPS fix 시각, ISO8601 UTC, **필수**) 와 `server_recv_ts`(서버 채움)를 분리하고 `ts_source: device_fix|server_recv` 플래그를 붙인다. `server_recv_ts − ts_fix` 지연이 임계 초과면 `is_estimate` 상향 — Tier0 배치덤프의 "순간이동 가짜궤적"을 차단.
- (d) **샘플링 간격 신고**: `sample_interval_s`(nominal)를 1급 필드로 POST한다. **"펌웨어 자유"의 조건 = "핑 간격을 신고할 것"** — 없으면 게이트에서 `raw`로 강등, 층화·가중 불가.
- (e) **점별 품질 공변량**: `fix_quality`(hdop 또는 sat_count) + `gnss_source ∈ {phone,l76k,other}`. 없으면 down-weight — 이질적 GPS 오차를 필터·가중.
- (f) **좌초 마커**: `motion_state: afloat|stranded|unknown`(서버 속도추정으로 자동판정 가능). stranded 점군은 표류 통계에서 제외/태깅 — 둔치 좌초 유닛의 정적 점군이 분산·이동 통계를 오염시키지 못하게.
- (g) `obs_id`(형식 `FF-<country>-<region>-<YYYYMMDD>-<seq>`)는 **서버가 부여**하고 그 id로만 인용(자체 생성 금지, 전역 유일성 보장).
- (h) `provenance.verification`은 정직하게: 실제 스튜어드/기관 확인 시에만 `steward_verified`, 아니면 `raw`/`auto_checked`(검증 마크 날조 금지).

**정직 가드**: `medium=freshwater`·`link=cellular`는 **설계 고정값**(위성 아님). 단일 트랙은 표본이므로 조립 트랙은 `is_estimate:true`.

### K2 · 회수-우선 (recover-first, count-in = count-out) — 정본 [PRELAUNCH_RECOVERY_CHECKLIST.md](../hardware/PRELAUNCH_RECOVERY_CHECKLIST.md)

**왜 절대 커널인가**: 우리는 하천 쓰레기를 *줄이려는* 프로젝트다. 방류한 드리프터를 강에 남기면 **우리가 바로 그 오염원**이 된다. 회수-우선이 흔들리면 프로젝트는 환경적 자기모순이 되고 지자체 점용·시민 신뢰·후원 서사가 동시에 무너진다.

**최소 계약**:
- (a) 방류 전 접근 클래스(A/B/C) 판정. 클래스에 맞는 회수 수단 확보 시에만 방류, **C=방류 금지**.
- (b) 회수 담당자·연락망·회수 마감(T+h) 사전 확정.
- (c) 세션별 정산 로그(`N_deploy·N_recover·N_lost·회수율`) 작성. 유실은 사유코드(`stranded·sunk·signal_lost·drifted_out·retrieved_damaged`)로 **투명 로그**(숨기지 않음).
- (d) **회수 교차검증**: `recover_status=recovered`는 **last_gps 종점 + 마지막 heartbeat 소거**를 증거로 요구. 증거 없는 `recovered`는 `unverified_recovery`로 표기(통계 위조 차단).
- (e) 배포 수 ≤ 회수 역량(대량 살포 금지, 단계적).
- (f) **Tier0(폰인병)은 테더 전용·Class A 전용**으로 못 박고, **PRELAUNCH 회수 게이트를 Tier0에도 하드 종속**한다(장벽0이 회수 면제가 아님).

**정직 가드**: 파열형 1차 리튬 금지·회수형 셀. 유실은 예외·보고 대상이지 정상이 아니다.

### K3 · 부력체 성능 통과 (Verified Float) — 정본 [FLOAT_STANDARD.md](../hardware/FLOAT_STANDARD.md)

**왜 커널인가**: "소재는 알아서 공수"가 신뢰성을 깨지 않게 하는 유일한 장치가 **통과/탈락 테스트**다. 소재 목록은 나라마다 깨지지만 성능 기준은 어디서나 유효하다. 가라앉거나 뒤집혀 안테나가 잠기면 GPS·셀룰러가 둘 다 죽고 회수(K2)·데이터(K1)가 동시에 무너진다. K3는 K1·K2를 물리적으로 떠받치는 층이다.

**최소 계약**:
- (a) **F1(소크: 24h 침수 질량증가 <5%)·F2(침수-부양: 하우징 침수 최악조건 72h 정립 부유) 둘 다 통과**해야 "Verified Float". 실측 수치는 전부 **추정(estimate)**, 빌드별 F1·F2로 검증. 실험실 불요(주방저울+양동이+시간).
- (b) 안테나는 수면 위·하우징 비금속(RF 투과)·자기복원(밸러스트로 안테나가 위로 서게).
- (c) **B8 · 압력균등/열관리(K3로 승격)**: 밀폐 빌드는 압력균등 벤트(Gore-Tex 등) OR 입증된 열마진 필수. 직사광 밀폐+무벤트 리튬 금지. — 유닛이 F1+F2를 통과하고도 "화염병"이 되지 못하게.
- (d) **B9 · 셀 안전(K3로 승격)**: **보호회로 18650 필수, 1차 리튬 금지, 밀폐빌드 셀-실패 격납 필수**(기존 회수문서에만 있던 규칙을 부력 커널로도 승격).
- (e) **B4 비독성 = 금지물질 하드리스트**(육안판정 아님): **금지 = 납/기타 중금속 밸러스트·무코팅 EPS비드·1차 리튬·침출성 연질 PVC·개방셀 흡수재.** 밸러스트는 **명시 허용목록**(무연 봉돌·스테인리스·모래캡슐 등)으로만.
- (f) **회수성 + 야생동물 안전**: 고시인성 색·ID/QR·회수 고리. 단 회수 루프는 **저프로파일 루프/파단 테더**로 조류·어류 얽힘 위험을 저감([DEPLOY_responsibly.md](DEPLOY_responsibly.md)·DISCLAIMER 요건).

### K4 · 정직 라벨 (honest labels) — 정본: 각 문서 정직가드 각주

**왜 커널인가**: 오픈소스 무브먼트·지자체·후원자의 신뢰는 정직에서만 나온다. "위성 추적"·"AI가 예측"·"완벽 방수" 같은 과장은 한 번의 반증으로 전체 데이터셋과 커뮤니티 신뢰를 오염시킨다. 라벨이 자유 껍질이면 각 빌더가 제각기 부풀려 데이터의 의미가 붕괴한다.

**최소 계약**:
- (a) 고정 어휘 유지: **셀룰러(위성 아님)·물리 계산(학습 AI 아님)·회수-우선(count-in=count-out)·모든 부력/질량/표류 수치=추정·독점/락인 언어 금지.**
- (b) **`link=cellular`는 고정값**. 위성 모듈(Swarm/Iridium 등) 유닛은 껍질 변형이 아니라 **Class3(다른 기기)** — Class1 데이터풀·ChangeX 브랜드·Friendly Floaty 이름 사용 금지.
- (c) **예측/정확도% 주장(온디바이스 ML)은 커널 밖**: FF 정직 프레이밍·`is_estimate` 사용 불가, 별도 라벨. 펌웨어는 자유이되 "AI 예측"을 FF 이름으로 마케팅하는 것은 껍질이 아니다.
- (d) `verification` 마크·수치는 실측/추정 구분 명시. 데이터는 CC BY로 열려 있고 자체 호스팅 가능하되(§5.4) "우리 것만 진짜/호환" 식 락인 언어 금지.

## 2. 자유 껍질 카탈로그 — 축별 경계

각 축은 **최대한 열려 있다.** 커뮤니티 DIY의 힘은 여기서 나온다. 모든 축은 딱 한 지점에서 커널과 만나며, 그 경계만 지키면 나머지는 전부 자유다.

| 축 (껍질) | 바꿔도 됨 (자유·권장 다양성) | 커널이라 지켜야 함 (경계) | 커널 |
|---|---|---|---|
| **하우징 소재** | 폐페트병·업사이클 용기·3D프린트 케이스·IP케이스·드라이백 등 무엇이든 | 비금속(RF 투과)·밀봉·안테나 수면 위·**F1/F2 통과**·밀폐면 압력균등 벤트(B8) | K3 |
| **부력재** | EVA폼·폐어구부표·수영누들·밀폐 HDPE/PP·밀봉 PET(업사이클 우선) | 폐쇄셀(F1 <5%)·예비부력 +20~25%·**금지물질 하드리스트 밖**·비탈락 | K3 |
| **컴퓨트/보드** | Tier0 재사용폰~Tier1 LILYGO~타 A76xx 보드~Tier2 견고빌드 | 실제 GNSS 탑재·셀룰러(위성 아님)·스키마대로 POST·**보호셀·1차리튬 금지(B9)** | K1·K3·K4 |
| **LTE 밴드/모뎀** | 지역 밴드 A7670 변종(E/G/SA)·현지 데이터SIM·글로벌 IoT SIM | **해당국 망 실제 attach 실측 검증**·형식승인(§5.4). 미검증 모듈로 데이터 공백 금지 | K1 |
| **폼팩터** | 병·오리(friendlyFloaty)·커스텀 마스코트·지역 상징물 | 자기복원·안테나 위·**저프로파일/파단 테더(얽힘 저감)**·ID/QR·고시인성·**형상 바꾸면 calibration_id fork(§5.1)** | K2·K3 |
| **펌웨어** | 언어·라이브러리·ping 주기·절전 로직·자체 기능 추가 자유 | 최소 페이로드를 스키마대로 POST·`schema_version`·`ts_fix`·`sample_interval_s` 포함. **ML 예측을 FF 정직프레임으로 주장 금지** | K1·K4 |
| **회수 방법** | 뜰채·도보·카약·보트(A) / 드론·하류 캐치포인트(B) | 회수는 "방법"이 아니라 "출입증": 계획·담당자·마감·정산·**교차검증 증거** 필수, C=방류금지 | K2 |
| **현지화** | 언어·문서 번역·후원/게임 서사·운영 모델(자원봉사/유상 혼합) | K1~K4 자체는 번역·현지화 대상이 아님. **무선 형식승인·SIM 로밍 합법성·수역 사인오프·드론 규제는 하드게이트(§4.4)** | K1~K4 |

> **경계 판별 한 줄**: 바꾸려는 것이 (1) 데이터를 못 합치게 하거나 (2) 회수를 어렵게/불가능하게 하거나 (3) 안 뜨거나 뒤집히거나 안테나를 잠기게 하거나 셀을 위험하게 하거나 (4) 주장을 부풀리면 — 그건 껍질이 아니라 **커널을 건드린 것**이다. 그 외엔 전부 자유.

### 3.1 티어 = 진입장벽 축 ("아래로 내려갈 자유는 항상 열림")

| 티어 | 무엇 | 도구·기술 | 납땜 | 비용(USD) | 캠페인 수명 | 누가 | 커널 4 적용 |

원칙: **"위로 갈 자유"만 아니라 "아래로 내려갈 자유"가 항상 열려 있다.** 조달·기술이 부족하면 언제든 Tier0로 내려가 같은 스키마에 합류한다.

### 3.2 도구 최소 빌드 3경로

- **3D프린터 없음**: 프린트 3부품(브래킷·밸러스트킬·회수고리)의 무프린터 대체 — 폼블록 홈·케이블타이·**무연 봉돌**(하드리스트 준수)·에폭시·파라코드. 단 **F2 자기복원 확인 필수**.
- **납땜 못함**: 기준 보드가 JST/uFL/USB 커넥터형이라 Tier0~1은 무납땜 완성. 납땜은 **Tier3 솔라 옵션에서만** 등장.
- **실험실 없음**: F1·F2는 **주방저울+양동이+시간**으로 수행되도록 설계됨.

### 3.3 스키마·POST 죽도록 단순

- 최소 페이로드는 사람이 외울 수준: `{device_id, lat, lon, ts_fix, sample_interval_s}`. `crs`는 기본 EPSG:4326.
- 한 줄 curl 예시 + GPSLogger 커스텀 URL(`%LAT/%LON/%TIME`)로 **Tier0은 코드 0줄**.
- **인용ID(obs_id)·provenance·calibration_ref·라이선스는 서버가 부여**(빌더 부담 0). `verification`은 정직 승격만 가능(날조 불가).

### 3.4 다양성 = 접근성 (같은 커널·다른 진입로)

| 행위자 | 자원 | 권장 진입로 | 합류 지점 |
|---|---|---|---|
| **학교** | 낮음·미성년 | Tier0 테더/Class A·성인 감독([SAFETY_MINORS.md](../SAFETY_MINORS.md)) | 같은 스키마 |
| **메이커** | 중간 | Tier1~2 | 같은 스키마 |
| **저소득 지역** | 낮음·조달 제약 | Tier0 폰인병·현지 업사이클 소재 | 같은 스키마 |
| **지자체/연구** | 높음 | Tier2~3 솔라·steward 검증 | 같은 스키마 |

**작은 고정 커널 + 넓은 자유 껍질**의 접근성 번역: 진입로는 자원 수준마다 다르되, **커널 4개가 같으므로 첫 방류부터 모두가 같은 데이터에 합류한다.**

## 4. 커뮤니티 다양성 메커니즘

> 배치 제안: §4.1·4.2·4.4 → `COMMUNITY.md`, §4.3 → `CONTRIBUTING.md`. 커널·껍질 정본은 재정의하지 않고 링크만 한다(중복 최소화).

### 4.1 변형 레지스트리 / 갤러리

**목적**: 각 커뮤니티가 자기 빌드를 공개·검색·리믹스한다(Precious Plastic how-to·RepRap 계보 방식). 중앙이 "정답 하나"를 배급하지 않고 다양한 실증이 서로를 인용·개량한다.

**★레지스트리 등록 ≠ 데이터 자격**: 등록은 누구나 자유롭게 한다. 그러나 등록 자체가 데이터셋 포함을 뜻하지 **않는다**. 데이터 자격은 오직 K2 provenance 게이트(§5)가 정한다. 레지스트리는 "이런 빌드가 존재한다" 카탈로그이고, 데이터는 별도 게이트를 통과한다. **이 분리가 자가등록이 데이터 오염 구멍이 되지 않게 하는 1차 방어선이다.**

**최소 등록 스키마** — 한 변형 = `registry/*.yml` 1개:

```yaml
# registry/kr-busan-hakjang-bottle-v1.yml
variant_id: "kr-busan-hakjang-bottle-v1"     # ● 전역 유일 <country>-<region>-<form>-<rev>
display_name: "학장천 업사이클 PET 병형"
operator: "gaa-kr-busan"                       # ● 등록 operator(개인 아님) — API key로 서명(§5)
region: { country: "KR", admin: "busan", water_body: "nakdong-hakjang" }
form_factor: "bottle"                          # bottle | duck | custom
housing_material: "upcycled PET + dry bag"     # 껍질(자유)
float_material: "waste buoy foam (EVA)"        # 껍질(자유)
compute:
  tier: "T1"
  board: "LILYGO T-A7670G R2 (L76K GNSS)"
  lte_band_variant: "A7670G"                   # ★지역 밴드 변형(현지화 핵심)
  sim: "1NCE global IoT"
firmware_ref: "github:.../ff-fw@v1.2 (schema 1.0 POST)"
calibration_ref: "ff-drifter-v1"               # ● geometry_hash에 바인딩(§5.1) — 형상 바꾸면 fork 강제
compat_selfcert:                               # ● §4.2 배지 자가선언
  schema_emit: true                            #   K1
  float_test: { F1_soak: pass, F2_swamp: pass, cell_safety: pass }   # K3(B8/B9 포함)
  recovery_rule: true                          #   K2
  honest_labels: true                          #   K4
  evidence:                                    # ● ★유닛별 유일·타임스탬프·해시(§5)
    - { type: F1F2_log, url: "https://.../F1F2.pdf", sha256: "…", ts: "2026-09-10T…Z", unit: "…" }
remix_of: null                                 # ● 부모 variant_id(리믹스 계보) — 부모 key 서명 필요(§5)
license: { hardware: "CERN-OHL-S (see LICENSE-hardware.md)", docs: "CC BY 4.0", firmware: "see LICENSE" }
results: "5기 방류, 회수율 100%(교차검증), 표류 4.2km(estimate)"
```

**갤러리**: 지역·폼팩터·티어·밴드변형·회수클래스·operator로 검색. fork→`remix_of` 기입→바뀐 껍질만 diff, 부모 operator 크레딧 승계. 카드에 `compat_selfcert`·`float_test`·증거 유무를 **그대로 노출**하고 증거 없는 선언은 `unverified` 뱃지로 시각 구분.

### 4.2 "Friendly Floaty Compatible" 자가인증 배지 (OSHWA식)

중앙 승인·수수료 없이 **커널 4개(전부 AND)를 자가선언**하면 배지가 붙는다. 게이트키퍼를 두지 않아야 글로벌로 퍼진다. **대신 자가선언이 "쓰레기 데이터·미검증 유닛" 구멍이 되지 않도록 §5의 안전장치를 구조로 박는다.**

배지 상태 3단계(갤러리 시각 구분):
- **`self-declared`**: 커널 4개 자가선언 + 유닛별 해시 증거 게시. 기본 Compatible.
- **`unverified`**: 선언은 있으나 증거 미첨부/미회수 세션. **경고 뱃지**, 데이터는 provenance 게이트 미통과.
- **`steward-verified`**: **독립 다수 스튜어드/관청**이 F1/F2·정산을 확인(§5 연합 조건). `steward_verified` 데이터를 emit할 수 있는 최상위. 중앙 승인이 아니라 분산·연합 스튜어드 확인이다.

### 4.3 기여 흐름 (CONTRIBUTING)

**껍질 기여는 자유·즉시, 커널 변경은 RFC + 버전업 + 하위호환.**

| 경로 | 대상 | 절차 | 리뷰 강도 |
|---|---|---|---|
| **변형(Variant)** | 새 껍질(하우징/부력재/폼팩터/밴드/펌웨어) | `registry/*.yml` 추가 PR | 가벼움 — 스키마 lint + 링크·서명 유효성 |
| **리믹스(Remix)** | 기존 변형 개량 | fork → `remix_of` + **부모 key 서명** | 가벼움 — 크레딧 승계 확인 |
| **개선 PR(Improve)** | 문서·CAD·펌웨어·계산식 | 리포 PR | 중간 — 정합·안전 리뷰 |
| **커널 변경(Kernel)** | 스키마 버전·F1/F2·회수 규칙·정직가드 | RFC 이슈 → 논의 → 표준 개정 | 엄격 — 메인테이너 승인 + 하위호환 + 버전 범프 |

**PR 커널 불변식 4문** (하나라도 "아니오"면 껍질 기여가 아니라 커널 RFC):
1. **데이터 비교가능성**을 깨지 않는가?(스키마 필드·`obs_id`·`sample_interval_s`·`ts_fix`·`calibration_ref`↔geometry 유지)
2. **회수-우선**을 깨지 않는가?(count-in=count-out·A/B/C·C=방류금지·교차검증 유지)
3. **안전(F2/B8/B9)**을 깨지 않는가?(침수부양·자기복원·셀 벤팅/열관리·1차리튬 금지·안테나 위 유지)
4. **정직 라벨**을 깨지 않는가?(셀룰러≠위성·물리계산≠학습AI·락인 금지·estimate 표기)

크레딧: `remix_of` 계보로 원저자 자동 승계(부모 서명 유효 시), 데이터는 `obs_id`로 CC BY 4.0 인용([schema.md](../data/schema.md) §6). 라이선스는 하드웨어/펌웨어/문서 **3원 분리**([LICENSE-hardware.md](../LICENSE-hardware.md)·[LICENSE-docs.md](../LICENSE-docs.md)).

**새 지역 온보딩 6단계**: (1) 밴드/SIM 확정 + attach 실측(위성 아님 확인) → (2) 무선 형식승인·SIM 로밍·수역 사인오프·드론 규제 **하드게이트 통과** → (3) 접근 클래스 재판정·회수 파트너(C=방류금지) → (4) 소재 공수 + F1/F2/B8/B9 통과 → (5) 레지스트리 등록 + Compatible 자가선언 + 유닛별 해시 증거 → (6) **커널 4개는 그대로** → 첫 방류부터 글로벌 허브와 상호운용.

## 5. 다양성이 넘지 말아야 할 선 — 가드레일 (loophole 차단)

> **핵심 원리: "가입"은 자가선언으로 열되, "데이터 자격"은 기계 게이트로 잠근다.** 배지는 커뮤니티 참여증이고 데이터 통과는 별개다. 거짓 배지 선언이 있어도 데이터는 게이트에서 걸러진다. 아래는 적대적 리뷰가 뚫은 구멍을 하나씩 닫는다.

### 5.1 비교가능성 가드레일 (K1을 조여 데이터가 실제로 합쳐지게)

- **게이트를 '존재검사' → '증거검사'로.** `verified_build`의 F1·F2·셀안전 항목은 운영자가 스스로 true로 POST하는 불리언만으로 통과되지 **않는다**. **유닛별 유일 해시가 앵커된 증거**(질량로그/사진/영상)가 있어야 `raw` 초과 등급으로 승격한다. 증거 없는 self-attested 유닛은 **풀에서 격리·시각 구분·down-weight**(동등가중 병합 금지) — "명예 게이트"가 아니라 증거 게이트.
- **evidence 위조 차단.** 증거는 **유닛별 유일성·타임스탬프·sha256 해시**를 요구한다. 스톡 사진·404·100개 유닛이 URL 1개 공유는 무효. **스팟체크 주기·주체·revocation 트리거를 수치로 명시**(예: 신규 operator 첫 N세션 100% 검토, 이후 무작위 X%).
- **샘플링 간격·시간축·정밀도**를 K1 최소계약(§1)에서 필수화 — 5분/60분 혼합, 배치덤프 순간이동, 좌표 절단, 데이텀 불일치를 스키마 차원에서 봉쇄.
- **calibration_ref ↔ geometry_hash 바인딩.** `calibration_id`에 폼팩터+밸러스트+흘수 사양의 `geometry_hash`를 묶는다. **형상 껍질을 바꾸면 반드시 새 calibration_id로 fork, 재사용 금지.** `validated_sites` 추가는 동일 geometry_hash에서만 — 병(bottle)과 오리(duck)가 같은 프로파일을 공유해 "이전가능 과학자산"이 붕괴하는 것을 차단.
- **motion_state**로 좌초 점군을 표류 통계에서 분리(§1-f).

### 5.2 무결성·쓰기인증 가드레일 (K2를 조여 데이터 포이즌·통계위조 차단)

- **operator 쓰기 인증.** `provenance.operator` 자유문자열 폐기 → **등록 operator + per-operator API key**. 미인증 POST는 **격리**(operator 검증 전까지 데이터 미반영). 브랜드 사칭·악의적 귀속(타 operator 평판 오염) 차단.
- **서버측 물리 타당성.** 좌표·시간·배터리 범위·중복(dedup)·**속도/순간이동 상한**을 자동 검사해 `auto_checked`로만 승격. GPS 스푸핑·조작 트랙을 거른다.
- **회수 교차검증.** `recovered`는 last_gps 종점 + 마지막 heartbeat 소거 증거 요구, 증거 없으면 `unverified_recovery`(§1 K2-d). "대량 방류→전량 recovered 표기→통계는 깨끗, 환경엔 투기"를 차단.
- **estimate 강제.** 단일 트랙은 `is_estimate:true`로 강등, `medium`/`link`는 설계 고정값이라 위성·학습AI 주장 유닛은 **스키마 자체로 탈락**(K4 자동 강제).

### 5.3 자가호스트 우회 가드레일 (커먼즈가 self-host 신뢰에 종속되지 않게)

- **self-host 업스트림 재게이트.** [schema.md](../data/schema.md) §5의 "자체 서버 OR 공유맵 POST"는 축복이되, **self-host의 `steward_verified`는 커먼즈 병합 시 신뢰하지 않는다**. 업스트림/머지 경로는 **게이트 재실행 + 외부 독립 steward 재확인**을 요구한다. 자가호스트가 자기 서버에서 전부 `steward_verified`로 찍고 인용 요청해도 커먼즈에서 강등·재검증된다.

### 5.4 안전·정직 가드레일 (껍질이 안전·정직을 못 깨게)

- **셀 안전을 부력 커널로.** B8(압력균등/열관리)·B9(보호 18650·1차리튬 금지·셀-실패 격납)를 K3에 흡수(§1). "F1+F2 통과했지만 화염병"을 물리적으로 봉쇄.
- **금지물질 하드리스트.** B4를 육안판정에서 하드리스트+허용목록으로(§1 K3-e). 납 밸러스트·무코팅 EPS·침출 PVC를 커널 차원에서 금지.
- **위성 재도입 방화벽.** `link=cellular` 고정값, 위성 유닛=Class3(다른 기기)로 Class1 풀·브랜드 차단(§1 K4-b).
- **학습AI 재도입 차단.** ML 예측/정확도% 주장은 커널 밖, FF 정직프레임·`is_estimate` 사용 불가(§1 K4-c).
- **야생동물 안전.** 회수 루프=저프로파일/파단 테더(§1 K3-f).
- **접근성 push의 미회수 리스크.** Tier0는 테더 전용·Class A 전용, PRELAUNCH 하드 종속(§3.1) — "장벽0"이 회수 면제가 되지 않게.

### 5.5 배지·상표·연합 가드레일

- **steward 연합(위장된 중앙통제 차단).** `steward_verified`가 GAA 단독 발급이면 그것이 곧 오픈 배지를 쓴 중앙통제다. **독립 다수 인증기관으로 연합**하거나, 아니면 "비중앙" 표방을 철회한다(정직).
- **계보 서명(크레딧 세탁 차단).** `remix_of`·`calibration_ref` 상속은 **부모의 operator key 서명으로만 유효**. 서명 없이 유명 변형/프로파일 이름만 달아 평판·과학자산을 상속받는 provenance 세탁 경로를 봉쇄.
- **상표=안전 조건부.** 열린 설계는 자유(라이선스), **이름·배지는 안전·회수 준수 조건부**이며 위반 시 브랜드 사용 취소권([TRADEMARK_POLICY.md](../TRADEMARK_POLICY.md) — Precious Plastic 모델). CC BY 데이터 라이선스는 안전을 단속할 수 없으므로 상표정책이 실효를 준다.
