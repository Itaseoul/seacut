> **한 줄 요약**: AI는 파라메트릭 CAD·질량/부력 예산·DOE·펌웨어를 빠르고 정확히 뽑지만 침수·낙하·현장 부유 물리 검증은 대체 불가이므로, 정답은 "AI가 스크립트로 설계·계산 → 저비용 물리테스트 → params로 되먹임"의 폐루프다.

## 0. 요지
MCP는 아직 성숙한 범용 CAD 서버가 없으므로, **스크립트가 곧 인터페이스인 도구(CadQuery/build123d·KiCad·OpenSCAD)** 로 AI가 코드를 직접 쓰게 하는 게 현재 가장 확실한 경로다.

## 1. AI가 잘하는 것 vs 못하는 것 (정직 구분)

### 잘함 — AI에 위임 가능
| 작업 | AI 강점 | repo 연결점 |
|---|---|---|
| 파라메트릭 스크립트 CAD | 텍스트→형상, 치수=코드 한 줄 | `hardware/cad/params.scad`(실존), `*.scad` |
| 질량/부력 예산 | 밀도×체적 반복 산술 | 신규 `hardware/sim/buoyancy.py` |
| 공차 스택 | `clr=0.4`·`wall=2.4` 누적오차 전파 | `params.scad` |
| BOM 원가·대체품 매트릭스 | 표 조합·지역 단가 | `BOM.md`, `docs/OPEN_HARDWARE_BOM_global.md` |
| DOE(설계실험) | 밸러스트 질량×위치, 폼 체적×위치 자동 스윕 | 밸러스트=튜닝변수 |
| 펌웨어 | TinyGSM-fork/TinyGPSPlus 보일러플레이트·POST 스키마 | `firmware/`, `data/schema.md` |

### 못함 — 물리 테스트로만 확정 (AI 금지 영역)
- **밀봉 누수/침수**: 마린 실리콘 시공 품질·장기 침지 누수는 실물 수조 24–72h(VERIFIED_BUILD T1/T2)로만 확정. 시뮬 날조 금지.
- **자기복원 실동작**: CG-CB는 계산으로 초기값만, 안테나가 실제 수면 위로 서는지는 수조 관찰(T4).
- **낙하/충격**: 실물 낙하시험(T3).
- **현장 부유 거동(windage/drag)**: 실제 하천 표면풍·표층류 표류계수는 **현장 GPS 트랙 대 OpenDrift 역보정**으로만 산출. AI가 사전 "정확도 %"를 말하면 안 됨(정직 가드).

> 원칙: AI 산출물은 전부 **"추정(estimate)"** 라벨. 물리 테스트가 그 추정을 승인·기각한다.

## 2. 연계 프로그램·역할

### 2-1. CAD (파라메트릭)
- **OpenSCAD**(현재 사용·유지): 순수 텍스트 CAD. AI가 이미 `params.scad`(bottle_id=90, board 68×34×12, cell 18.6×68, ballast 55×22) 편집 가능. 단순 브래킷/킬/고리에 충분. 단점=유기 곡면·필렛·조립 구속 약함.
- **★CadQuery / build123d (Python) — AI 코딩 최적, 강력 추천**: 파이썬 API로 형상 생성 + **같은 스크립트에서 질량·체적·무게중심 계산** → 부력 예산과 한 파일 폐루프. STEP/STL 내보내기, 필렛·쉘 우수. PET 병 인서트를 이식하면 "형상+질량+부심"이 한 스크립트에.
- **FreeCAD (+FreeCAD-MCP)**: GUI 검수·도면·어셈블리용. FreeCAD-MCP 대화형 편집은 실험적 → 데모 수준, 파이프라인 신뢰 축으로 삼지 말 것.

### 2-2. 전자/PCB
- **KiCad**: Python 스크립팅·CLI로 AI 친화적. Tier 1은 완제품 LILYGO라 PCB 불필요. **Tier 3 캐리어 보드**(전원 안정화·슈퍼캡·NTC 충전보호·~2A 피크 완충)로 갈 때 BOM/네트/거버 자동화.

### 2-3. 시뮬레이션 (경량 우선)
- **정수역학 부력·자기복원 = 소형 파이썬 스크립트로 충분**(아르키메데스·CG-CB 비교, 수십 줄). **무거운 CFD 불필요**.
- **OpenFOAM은 옵션(고비용)**: 낙수·와류 동적 유동 정량화 시에만. 우리 목적(뜨냐/서냐/침수 시 버티냐)엔 과잉 — 기본값은 넣지 않음.
- **열/전원 예산 = 스프레드시트/스크립트**: 모뎀 송신 피크 ~2A(BOM.md 명시) 전압강하, 저수온 충전 컷오프 표 계산.

### 2-4. 표류 물리
- **OpenDrift**(이미 사용): 기구체 **windage/drag 보정계수** 산출 엔진. 현장 GPS 트랙을 입력으로 파라미터 **역추정(fit)**. (1.14 config 주의는 기존 메모대로.)

## 3. ★추천 파이프라인 — 폐루프

```
[AI 파라메트릭 설계·계산]                     [저비용 물리테스트]
 CadQuery/build123d 형상 + 질량/부심   ──►  ① 부력·자기복원 수조시험(T2/T4)
 파이썬 부력·자기복원 스크립트                ② 밀봉 침수 24–72h(T1/T2)
 (params 값 = 단일 진실원천)                  ③ 낙하 투하시험(T3)
        ▲                                    ④ 현장 표류 GPS 트랙
        │                                          │
        └──────── params.scad / params.py ◄────────┘
                  (측정치를 변수로 되먹임)
```

### repo 붙는 지점(구체)
1. **단일 진실원천**: `hardware/cad/params.scad` 는 **실존**하며 `bottle_id/wall/clr/board_*/cell_*/ballast_*` 정의됨. 이를 **동일 이름 `params.py`** 로 미러링 → CadQuery·부력 스크립트·OpenSCAD 공유. 밸러스트·폼은 여기서 튜닝. ★폼 칼라 치수(Ø86·채널70×40)는 params에 **없는 신설 부품**이므로 새 변수로 추가.
2. **부력 예산 스크립트** `hardware/sim/buoyancy.py`(신규): params 로드 → (a) 밀봉 예비부력비=V_disp/M, (b) **침수 순부력**, (c) 자기복원 여유 출력. DOE 모드로 밸러스트 질량·폼 체적 스윕(트레이드오프 ΔV_foam=(1.25·Δm − Δm/ρ_ballast)/(1−ρ_foam) 반영).
3. **테스트 프로토콜 문서**: 이제 `hardware/RELIABILITY.md`·`VERIFIED_BUILD_TEST.md`·`PRELAUNCH_RECOVERY_CHECKLIST.md` 신설 완료 → 각 테스트 합격기준을 params 되먹임 규칙과 함께 고정.
4. **되먹임 규칙**: 침수시험 실패→폼 체적↑, 자기복원 실패→`ballast_l/w`·밸러스트 질량↑(+폼 트레이드오프), 표류 편차→OpenDrift windage 계수.

### 계산이 이미 확정하는 것 (설계 사실, RELIABILITY.md와 통일)
- 밀봉 1L 병: 배수 ~1000cm³ → 담수 부력 ~1000gf. 총질량 **M_wet ≈ 207g**(밸러스트 40g 기준) → **예비부력비 ~4.83×(부력/질량)**. **멀쩡할 땐 안 가라앉음** — 계산 확정.
- **진짜 실패모드=침수**: 병이 물차면 공기 상실, 고형부품만 자기 체적 부력 → **순부력 −97gf → 침몰**(배터리 −29.5·밸러스트 −34.9 지배).
- **대책=폐쇄셀 폼 1차 부력**: 정본 1.25 예비부력비식(고형 자체배수 101.9cm³ 포함) → **≥155cm³ → 채택 160cm³(6.4g)**, 침수 예비부력비 1.275. ★고형 자체배수를 무시한 245cm³식은 과대(안전율 상향 옵션으로만). 1L 병 상부 공간 수납 가능. **"침수돼도 뜨는" 설계 = BOM에 폐쇄셀 폼 신설**이 핵심 개선(회수-우선과 정합).

## 4. MCP 관점 (과장 없이)
- **현실**: 성숙한 범용 CAD-MCP는 아직 제한적. "AI가 MCP로 임의 CAD를 자유 조작"은 프로덕션 신뢰 축이 아니라 데모 단계.
- **그래서 스크립트=인터페이스 도구가 정답**: CadQuery/build123d·KiCad·OpenSCAD는 **코드 자체가 API**라 별도 MCP 없이 AI가 파일 직접 편집·CLI 실행·검증하는 방식이 가장 안정적(이 환경에서 이미 성립 — Read/Edit + Bash 렌더·계산).
- **FreeCAD-MCP**: 대화형 GUI 편집 탐색은 하되, 신뢰 경계는 **재현 가능한 스크립트+물리테스트**에 둔다. MCP는 편의, 검증은 물리.

## 5. 바로 실행 가능한 다음 스텝
1. `hardware/cad/params.py` 신설 — 실존 `params.scad`와 변수 1:1 미러(단일 진실원천).
2. `hardware/sim/buoyancy.py` 신설 — 정상 예비부력비/침수 순부력/자기복원 여유 출력, `--sweep`로 밸러스트·폼 DOE.
3. **CadQuery 이식 PoC** — 인서트 1종(예: `ballast_keel`) 재작성해 "형상+질량+부심" 한 스크립트 검증(성공 시 나머지 이식).
4. **BOM에 폐쇄셀 폼 라인 추가** — Tier 1에 "1차 부력 폐쇄셀 폼 160cm³(신설 부품)" 추가, "침수돼도 뜨는" 설계로 승격.
5. RELIABILITY/VERIFIED_BUILD/CHECKLIST 문서 유지·갱신 — 부력/침수/낙하/표류 프로토콜 + 합격기준 + params 되먹임 규칙.
6. **OpenDrift 역보정 스크립트 스텁** — 현장 GPS 트랙 입력 시 windage/drag 계수 fit(현장 데이터 확보 후 실행, 사전 수치 날조 금지).
7. (선택) FreeCAD-MCP 30분 탐색 — 대화형 편집 가능성만 확인, 파이프라인 신뢰 축엔 미편입.
8. **모든 산출 수치에 "추정" 라벨+계산식 병기** — 정직 가드 상시 적용.

---
> **정직 가드 각주**: 셀룰러(LTE, 위성 아님)·담수(ρ=1.0) 기준·물리 정수역학 계산(학습형 예측·정확도% 아님)·회수-우선·현장 데이터 날조 금지. 예비부력비=부력/질량으로 정의 통일. 모든 수치는 **추정(estimate)** 이며 계산식과 함께 제시.
