# hardware/ELECTRONICS_POWER.md — 전원 예산 · 전원 하드웨어 (Friendly Floaty / SEA:CUT)

**한 줄 요약:** Class 1 하천 회수형(우리 Tier1 BOM = LILYGO T-A7670G R2 + 보호 18650)의 전원 예산은 **핑 간격과 "모뎀 등록유지 vs 모뎀 오프"라는 설계 분기**에 지배되며, 상온 기준 5분 ≈ 1.6~4일 · 15분 ≈ 3.5~7.4일 · 60분 ≈ 6.3~19일 수준으로 **추정**된다(전부 벤치 실측으로 확정 필요). 통신은 **셀룰러(위성 아님)**, 운영 대원칙은 **회수-우선(count-in = count-out)**, 태양광은 **기본이 아닌 Tier-3 옵션**이다.

---

## 0. 정직 기준 (먼저 고정)

- **모든 전력 수치는 추정(estimate)이며 넓은 범위 + "벤치 실측 필요"다.** 전류 프로파일러/파워 애널라이저로 실기기 실측하기 전까지 어떤 수치도 확정으로 인용하지 않는다. 특히 **슬립 전류·GPS 콜드픽스 에너지·LTE attach 평균전류**는 편차가 커서 사이클당 소비의 지배 변수이자 최대 불확실 요인이다.
- **통신은 셀룰러(LTE Cat.1 bis, A7670G)다. 위성 아님.** 셀 커버리지가 없는 외해에서는 이 기기가 작동하지 않으며, 그것은 **Class 3(다른 기기 클래스)**의 영역이다. ChangeX(부산 소하천 PoC) 제출·예산·설계에는 **Class 1만** 등장한다.
- 위치·상태는 **물리 계산·측위**이지 학습형 AI 예측(정확도%)이 아니다.
- 배터리는 **보호회로 포함 18650**, 가용 용량 **~2500 mAh @ 공칭 3.7V (≈9.25 Wh)**로 **가정**한다. 보호 PCB 소비·자기방전·저온/컷오프 여유 때문에 데이터시트 정격보다 보수적으로 잡은 값이며, **실측으로 교체해야 한다.**
- 태양광은 **친환경 기본값이 아니다.** Class 1 단기 회수형엔 대체로 순손해이며, **무인 + 주 이상 + 회수 불가**의 세 문턱을 동시에 넘는 Class 3에서만 켜는 옵션이다(§3).

## 1. Class 1 전원 예산 (기본 · ChangeX · 우리 BOM)

### 1.1 전제 · 가용 용량

| 항목 | 값(추정) | 근거 / 주의 |
|---|---|---|
| 가용 용량 `C_usable` | ~2500 mAh @ 3.7V (≈9.25 Wh) | 보호 18650. 보호PCB·자기방전·컷오프 여유 반영. 데이터시트 정격보다 보수적. **실측 필요** |
| 전원 급전 | **VBAT 직결 우선** | 모뎀은 VBAT 직결이 기본. ESP/센서용 3.3V LDO/부스트 경로가 있으면 ~10~15% 변환 오버헤드가 붙으므로 **직결 여부를 회로에서 1줄로 확정**할 것 |

### 1.2 계산식

```
사이클수/일        N        = 1440 / 핑간격(분)
사이클당 소비      E_cycle  = E_gps + E_esp + E_lte            [mAh]
일 소비            D        = (I_sleep[mA] × 24) + (N × E_cycle) [mAh/일]
가동일수(상온)     Days     = C_usable / D        (C_usable = 2500 mAh)
가동일수(냉수)     Days_cold = Days × k_cold       (k_cold = 0.70 ~ 0.90)
```

### 1.3 구성 요소 추정 범위 (★넓게, 벤치 실측 필요)

- **① 슬립 전류 `I_sleep`** — 이 보드는 초저전력 설계가 **아니다.** 온보드 LDO 정지전류 + 전원 LED + 모뎀 슬립 잔류가 합산된다. **슬립은 독립 변수가 아니라 모뎀 정책과 커플링**된다(§1.4).
  - 모뎀 **등록유지** idle: 실측 **5~15 mA**가 흔함
  - 모뎀 **완전 오프** + LED/불필요 LDO 차단: **1~3 mA** 목표 (★현 스톡 보드엔 없는 하드웨어 개조 전제)
- **② GPS fix** — L76K. 콜드 30~90s @ ~30~40 mA면 **~1.5 mAh/사이클**, 웜/핫이면 수 초로 **~0.2 mAh**. **핑 간격에 의존**(§1.7).
- **③ ESP 활성** — 처리·대기 ~40~80 mA, 10~30s → **0.1~0.5 mAh/사이클**.
- **④ LTE attach + POST** — **순간피크 ~2A**(짧은 버스트), 평균 100~300 mA. **재attach(콜드 등록) 여부에 크게 좌우** → 등록유지면 낮고, 모뎀 오프면 매 웨이크 콜드 재attach로 커진다.

### 1.4 두 개의 정합 시나리오 (슬립 ↔ attach 커플링 명문화)

**낙관/중앙/비관을 뒤섞으면 자기모순이 난다**(최저 슬립=모뎀오프인데 최저 attach=등록유지를 동시에 못 씀). 그래서 **모뎀 정책 두 분기**로 예산을 재구성한다:

| 분기 | 슬립 `I_sleep` | 사이클당 `E_cycle` | 유리한 구간 |
|---|---|---|---|
| **A — 모뎀 등록유지 (keep-alive)** | 높음 **8 mA** | 낮음 **1.5 mAh** (웜 attach·웜/핫 GPS) | **단주기**(사이클이 잦아 사이클당 비용이 지배) |
| **B — 모뎀 완전 오프 (power-cycle)** | 낮음 **2 mA** | 높음 **3.5 mAh** (콜드 재attach·콜드 GPS) | **장주기**(사이클이 드물어 슬립이 지배) |

> **핵심 물리:** 두 분기는 **핑 간격에서 교차**한다. 단주기(5~15분)는 A가, 장주기(60분)는 B가 유리하다. 어느 쪽이 정답인지는 배포 주기가 정한다 — 단일 "낙관치"는 존재하지 않는다.

### 1.5 가동일수 표 (상온, `C_usable` = 2500 mAh)

| 핑 간격 | N | **A: 등록유지** (슬립8mA·1.5mAh/cyc) | **B: 모뎀오프** (슬립2mA·3.5mAh/cyc) | 권장·상온 범위 |
|---|---|---|---|---|
| **5분** | 288 | 192+432=624 → **약 4.0일** | 48+1008=1056 → **약 2.4일** | **A** 권장 ≈ **4.0일** |
| **15분** | 96 | 192+144=336 → **약 7.4일** | 48+336=384 → **약 6.5일** | ≈ 교차점, **A** 근소우위 ≈ **7.4일** |
| **60분** | 24 | 192+36=228 → **약 11.0일** | 48+84=132 → **약 18.9일** | **B** 권장 ≈ **18.9일** |

**비관(worst) 하한** — 스톡 보드(전원 LED 상시 + 고 idle 등록) + 콜드 픽스·콜드 재attach가 겹칠 때: 슬립 12 mA · `E_cycle` 4.5 mAh 가정 →

| 핑 간격 | 계산 | 비관 하한 |
|---|---|---|
| 5분 | 288+1296=1584 → 2500/1584 | 약 **1.6일** |
| 15분 | 288+432=720 → 2500/720 | 약 **3.5일** |
| 60분 | 288+108=396 → 2500/396 | 약 **6.3일** |

**정직 범위(상온, 비관 하한~권장 상한):** 5분 ≈ **1.6~4.0일**, 15분 ≈ **3.5~7.4일**, 60분 ≈ **6.3~18.9일**.

> ★ **이전 초안의 "60분 최대 52일"은 폐기했다.** 그 값은 슬립 1 mA(전원 LED 제거 + 모뎀 완전 오프)라는 **현 BOM에 없는 하드웨어 개조**를 전제하고도, 모뎀 오프의 콜드 재attach 비용을 낮은 attach와 섞어 과대 산출한 수치였다. **스톡 보드 무개조 현실은 60분에서 대략 6~19일**이며, 공격적이지만 현실적인 저전력 개조(LED 제거·LDO 차단·슬립 최적화)를 다 해도 **~25일 상단**이 정직한 천장이다. 52일은 인용 금지.

### 1.6 냉수 보정 + 저온 펄스-컷오프 여유

겨울 하천/냉수(≈5~10℃)에서는 셀 **가용 용량이 대략 10~30% 감소**한다. 권장 상온 일수에 `k_cold = 0.70~0.90`을 곱한다:

| 핑 간격 | 권장(상온) | 냉수 보정 후 |
|---|---|---|
| 5분 | 4.0일 (A) | 약 **2.8~3.6일** |
| 15분 | 7.4일 (A) | 약 **5.2~6.7일** |
| 60분 | 18.9일 (B) | 약 **13.2~17.0일** |

> ★ **`k_cold`는 순수 용량 감소만 반영한다.** 저온에서는 셀 **내부저항↑ → LTE ~2A 펄스 시 전압 새그↑ → 보호 PCB 조기 컷오프**가 겹칠 수 있어, 실사용 가용치가 `k_cold` 하한(0.70) **아래로 더 떨어질 수 있다.** 이는 용량 derate와 **별개의 여유**로 다루고, 저온 배포 시 핑 간격을 늘려 마진을 확보한다.

### 1.7 GPS 픽스 상태의 핑간격 의존 (콜드 균일적용의 방향성·한계)

에페메리스 유효기간(~2~4h) 때문에 픽스 상태는 핑 간격에 의존한다: **단주기(5~15분)는 대부분 웜/핫 픽스(수 초)**, **장주기(수 시간+)는 콜드 픽스가 지배적**이다. 위 표는 편의상 보수적으로 콜드 쪽을 얹었으므로 **단주기 열은 보수(과대 소비) 방향, 장주기 열은 콜드 정합**이다. 콜드픽스 "지배"는 장주기에서 성립하는 조건부 명제이며, 단주기에서 콜드 1.5 mAh를 매 사이클 적용하는 것은 안전측 과대 추정임을 유의한다.

## 2. 미션/기기 클래스 매트릭스

| 항목 | **Class 1 하천 회수형** (기본·ChangeX·우리 BOM) | **Class 2 하구·연안** | **Class 3 외해 라그랑주** |
|---|---|---|---|
| 미션 | 도시 소하천 병목→표류 실측, 대시민/교육 | 하구·연안 표류 추적, 캐치포인트 유입 관측 | 외해 라그랑주 표류·해양물리 과학계기 |
| 기간 | 시간 ~ 며칠 | 며칠 ~ 주 | 주 ~ 개월 |
| 전원 | **보호 18650 단독, 회수해 재충전** | 큰 배터리(2~4셀/대용량) ± **소형 태양광** | **태양광+충전** 또는 **대용량 1차전지(비충전)** |
| 통신 | **셀룰러 (LTE Cat.1 bis)** | **셀룰러** (연안 커버리지 내) | **위성** (셀룰러 커버리지 없음). ★우리 BOM 아님 |
| 회수 | **회수-우선** (전량 회수·재충전) | 캐치포인트/드론/보트 회수 | **사실상 회수 불가** → 저영향 소재로 손실 최소화 |
| 소재/환경 | 재활용 PET 캡슐 + 폐쇄셀 폼, 담수 정수역학 | 기수(ρ≈1.005~1.02), 생물부착·염수부식 대응 | 저독성·저영향 소재(OpenMetBuoy/GDP식) |
| 우리 위치 | **이 기기가 우리 것. ChangeX = 여기만** | 확장 후보(별도 설계·별도 BOM) | **다른 기기 클래스. ChangeX와 절대 혼용 금지** |

**핵심 프레임 한 줄:** 태양광·위성이 "필수"가 되는 순간은 **무인 + 주 이상 + 회수 불가**가 동시에 성립하는 **Class 3뿐**이다. Class 1(우리 BOM)은 사람이 회수해 재충전하므로 그 문턱을 넘지 않는다.

**회수-우선의 클래스별 강도:** Class 1은 "버리지 않는다"가 절대 규칙(방류 대수 = 회수 대수 장부 대조), Class 2는 "회수 경로를 설계에 내장"하는 목표치, Class 3는 회수가 물리적으로 불가능하므로 규칙이 **"소재로 방어"(저독성·저영향·손실 최소화)**로 치환된다. **우리 BOM 기기(Class 1)와 ChangeX는 이 치환을 하지 않는다 — 언제나 전량 회수다.**

## 3. 태양광 옵션 스펙 (Tier-3)

태양광은 **Class 3(무인·주 이상·회수 불가)** 또는 Class 2의 장기 무인 변형에서만 검토한다. Class 1 단기 회수형엔 **대체로 순손해**다. 아래는 "굳이 켤 때"의 조건부 스펙이다.

> ★**장기 잔류 드리프터 변형(Tier 1.5)**: 커버리지 내에서 주+ 잔류하는 드리프터에 솔라를 얹는 별도 변형은 [TIER1_5_SOLAR_DRIFTER.md](TIER1_5_SOLAR_DRIFTER.md)에 있다(원통 셀로 자기복원 보존·저온컷오프·벤치 G5 수확 검증). 본 §3의 고정 Tier-3와 구분되며, **단기 회수형엔 여전히 순손해**라는 위 원칙은 그대로다. 솔라는 전력을 늘리지 커버리지를 늘리지 않는다.

### 3.1 켜기 전에 통과해야 할 문턱 (셋 다 성립해야 검토)

1. **무인** — 사람이 접근·회수해 재충전할 수 없다.
2. **주 이상** — 배터리 단독 예산(~2500 mAh)으로 못 버티는 기간이다.
3. **회수 불가** — 손실을 전제로 저영향 과학계기로 설계한다(회수-우선 포기 → Class 3 소재 기준으로 전환).

### 3.2 부품·설정 스펙 (옵션 채택 시)

| 요소 | 스펙 / 조건 | 정직한 트레이드오프 |
|---|---|---|
| 소형 패널 | 세장형 수직 캡슐 상부에 소면적. **자기복원 시 패널이 하향**할 수 있어 발전량 불안정. **하천 탁도·수막·이물로 효율 추가 저하** | 방향·오염 손실을 크게 잡아 보수적으로 산정 |
| 충전 컨트롤러 | MPPT/PWM + **저온 충전 컷오프(0℃ 이하 리튬 충전 금지, JEITA/NTC) 필수** | 컷오프 없으면 저온 충전으로 셀 열화·안전 문제 |
| 밀폐함 열관리 | 패널·흑체 흡열로 밀폐 병 내부 온도↑ → **F2(셀 열폭주·벤팅) 악화.** 차광·통기벤트(Gore-Tex)·압력균등화 필수 | 태양광이 오히려 최악 실패모드를 키운다 |
| 추가 질량 → 부력 | 패널·컨트롤러·배선 추가 질량 = **부력 재계산 필수.** RELIABILITY 정본식 `[Σ고형배수 + V_foam] ≥ 1.25 × (M_dry + ρ_foam·V_foam)`으로 폼 재사이징 | 질량 늘면 흘수↑·건현↓·자기복원 여유↓ |
| 자기복원 상호작용 | 패널을 상부에 두면 CG 상승 위험(CG < CB 유지 필요) → 하부 밸러스트 재튜닝 → 밸러스트↑ → 다시 폼↑ (연쇄 재계산) | CG 상승은 복원성 직접 잠식 |

> **부력 재보정 계수(참고, ★정본 의존):** RELIABILITY §5 밸러스트-폼 트레이드오프식에 따르면 질량 +Δm에 대해 **폼 약 +11.7 cm³/10g** 수준의 재보정이 필요하고, 폼 160 cm³ 기준 침수 예비부력비 **1.275(PASS, 목표 ≥1.25)**의 여유가 태양광 추가로 잠식된다. ★이 교차인용 수치는 본 문서 내에서 검증 불가하며 **RELIABILITY 정본의 실재·정합을 별도 확인**해야 한다. **패널은 밀도가 달라 계수는 실측으로 재산정**해야 한다. (★저밀도 랩 필름 변형(Tier 1.5)은 밀도인지 **+6.7 cm³/10g** — [TIER1_5 §5](TIER1_5_SOLAR_DRIFTER.md). 여기 11.7은 리지드 패널 프레임 등 고밀도 추가물에만 해당.)

### 3.3 태양광 대신 대용량 1차전지가 나은 때

**저온·저일사·고탁도**(겨울 하구, 흐린 계절, 탁수)에서는 태양광 회수 에너지가 불확실하다. 이때는 **대용량 비충전 1차전지**가 (a) 충전 컷오프·과열 리스크 없이 (b) 예측 가능한 에너지 예산을 준다. 단, 1차전지는 **회수 불가 시 환경 유실 부하**가 크므로 **저독성·저영향 셀 선정 + 손실 최소화**(Class 3 소재 원칙)를 전제로만 쓴다. 회수 가능한 시나리오면 애초에 Class 1(18650 재충전)이 정답이다.

**결론(정직):** 태양광은 "친환경적으로 보이는 기본값"이 아니다. Class 1에선 **질량·과열·부력·복원성 네 축을 모두 악화**시키는 순손해이고, **무인 + 주 이상 + 회수 불가** 세 문턱을 다 넘을 때만 켜는 Tier-3 옵션이다.

## 4. 전원 하드웨어 노트 (설계 필수)

- **2A 피크 대응(벌크캡/슈퍼캡):** 18650 단독으로 LTE 송신 피크(~2A)를 때리면 셀 내부저항 때문에 전압이 순간 강하해 모뎀 브라운아웃/리셋 위험. **모뎀 전원 레일에 벌크 커패시터(470~1000 µF 이상) + 슈퍼캡/대용량 세라믹 뱅크를 병렬**로 두어 피크를 흡수한다. **USB 5V 단독 급전 금지**(USB 전류제한·케이블 강하로 attach 시 리셋 유발).
- **★보호 PCB 과전류 트립(캐패시터만으론 불충분):** ~2A 버스트가 **저전류 보호보드의 OCP**를 건드리면 리셋 루프에 빠진다. **연속 3A+ 정격의 보호 PCB**를 반드시 선정할 것. 벌크캡·슈퍼캡은 순간 새그를 완화할 뿐 OCP 트립 자체를 막지 못한다.
- **냉수 = 이중 손해:** 용량↓ + 내부저항↑ → 저온에서 2A 피크 시 전압 강하가 더 심해진다(§1.6 펄스-컷오프 여유). 방수함 내부에 셀을 두어 자기발열을 살짝 활용하거나, 저온 배포 시 핑 간격을 늘려 마진을 확보한다.
- **★저온 회수-재충전 회전율(운용 각주):** 0℃ 미만에서는 **리튬 셀 충전 자체가 금지**(저온 충전 = 셀 열화·석출)다. 이는 런타임이 아니라 **회수 후 재충전 회전 주기**에 영향을 준다 — 겨울 현장 회수 셀은 상온 복귀 후 충전해야 하므로 **회전율이 떨어진다.** 여분 셀 풀을 확보해 방류-회수 사이클을 유지한다.
- **슬립 최적화가 장주기 시나리오의 관건:** 60분처럼 사이클이 드물면 **슬립 드레인이 전체 소비를 지배**한다(§1.4 분기 B). LED 제거·불필요 LDO 차단·모뎀 완전 오프(재attach 비용 감수) 여부가 6일↔19일을 가른다 → **슬립 전류를 최우선으로 실측**할 것.
- **전원 변환 경로 확정:** 모뎀 VBAT 직결이면 변환 손실은 무시 가능하다. 그러나 **ESP/센서용 3.3V LDO나 부스트 경로가 있으면 ~10~15% 오버헤드**가 붙는다. 회로에서 직결 여부를 명시하고, LDO 경로가 있으면 위 예산에 여유율을 반영한다.
- **기성 보드 / Tier-3 KiCad 경계:** Tier1은 **커스텀 PCB 없이 기성 LILYGO T-A7670G R2**를 그대로 쓴다(개조는 LED 제거·LDO 차단·전원 레일 캡 추가 수준). 커스텀 KiCad 설계는 **Tier-3(장기 무인·태양광 옵션) 전용**으로 미룬다 — Class 1 회수형은 기성 보드로 충분하며, 커스텀 보드는 검증 부담만 키운다.

## 정직가드 각주

1. **추정·벤치필요:** 본 문서의 모든 mAh·일수·전류·피크·계수는 **추정 범위**이며, 특히 **슬립 전류(§1.3·§1.4)·GPS 콜드픽스(§1.7)·LTE attach 평균전류·저온 펄스 컷오프(§1.6)**는 **실기기 벤치 실측으로 반드시 교체**해야 한다. 정밀 주장으로 인용 금지.
2. **셀룰러 = 비위성:** 우리 BOM 기기는 셀룰러(LTE Cat.1 bis)다. 위성이 필요한 것은 **Class 3(외해·다른 기기)**뿐이며, **ChangeX와 절대 혼용하지 않는다.**
3. **물리 계산 = 비학습:** 위치·상태는 정수역학·측위 계산이며 학습형 AI 예측(정확도%)이 아니다. 운영 대원칙은 **회수-우선(count-in = count-out)**.
4. **태양광 = Tier-3 옵션:** 태양광은 기본이 아니다. **무인 + 주 이상 + 회수 불가** 세 문턱을 동시에 넘는 Class 3에서만 켜며, Class 1에선 질량·과열·부력·복원성 네 축을 악화시키는 순손해다.
5. **교차인용 주의:** §3.2의 예비부력비 1.275·+11.7 cm³/10g·reserve 1.25는 **RELIABILITY 정본 의존** 수치로 본 문서 내 검증 불가 — 정본 실재·정합 여부를 별도 확인하고, 패널은 밀도가 달라 계수를 실측 재산정할 것.
