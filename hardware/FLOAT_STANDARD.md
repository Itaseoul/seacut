# 부력체 표준 (Buoyancy Body Standard) — 자가 공수·성능 기반

> **한 줄 요약**: 우리는 부력체를 배급하지 않는다. 부력체가 갖춰야 할 **성능 조건과 그것을 증명하는 두 개의 현장 테스트**를 배급한다. 빌더는 현지에서 아무 부력체나(가급적 업사이클) 공수해, 이 표준을 통과시켜 "Verified Float"로 만든다.

## 0. 원리 — 소재가 아니라 스펙을 표준화한다

소재 목록은 나라마다 조달·가격이 달라 깨진다. **성능 기준은 어디서나 유효하다.** "알아서 공수"가 신뢰성을 해치지 않게 하는 열쇠는 **통과/탈락 테스트**다.

이는 Precious Plastic 방식이다 — 다양성을 허용하되 **스펙으로 경계 짓는다**(PP는 HDPE/PP 종류·기계 공차·안전 가이드로 경계). **무한정 다양 = 불신뢰.** 소재는 자유, 자격은 테스트.

## 1. 요구사항 — 부력체가 해야 할 것

| # | 요구 | 판정 |
|---|---|---|
| B1 | **침수 부양** — 전자 하우징이 완전 침수된 최악 조건에서도 조립체가 똑바로 뜬 채 유지 | 테스트 F2 |
| B2 | **폐쇄셀·비흡수** — 24h 잠겨도 질량 증가 **< 5%** (부력이 시간 따라 죽지 않게) | 테스트 F1 |
| B3 | **예비부력 여유** — 침수 시 순부력 **≥ +20~25%**(간신히 뜨는 수준 금지). 폼 사이징은 [RELIABILITY.md](RELIABILITY.md) §4 | 계산+F2 |
| B4 | **비독성·비탈락** — 강에 녹거나 부스러기(미세플라스틱)를 흘리지 않음. 부스러지는 EPS 알갱이는 코팅/밀폐 시만 허용 | 육안·소재이력 |
| B5 | **RF 경로** — 부력체가 안테나를 금속·물로 감싸지 않고, 안테나가 수면 위 유지(=벨리 밸러스트 자기복원). 설계검토 계산 [SELF_RIGHTING.md](SELF_RIGHTING.md)(GM=R−KG>0, 기본 +11mm), 실증은 T4/T7 | 조립검사+계산 |
| B6 | **회수성** — 고시인성 색 + ID/QR 부착 + 잡을 고리/그립 | 조립검사 |
| B7 | **캠페인 내구** — 방류 기간을 버팀(단기=PET 허용, 장기·저온=EVA/HDPE) | 소재 선택 |

## 2. 두 개의 현장 테스트 (실험실 불필요)

[VERIFIED_BUILD_TEST.md](VERIFIED_BUILD_TEST.md)의 항목과 연동한다.

- **F1 · 소크 테스트 (소재 자격)** — 부력체만 무게 측정 → 24h 침수(눌러 잠금) → 재측정. **질량 증가 < 5% 면 폐쇄셀 합격.** (하우징 밀봉의 누수 시험 T1과 인접하나, 이건 *부력재 자체의 흡수*를 본다.)
- **F2 · 침수-부양 테스트 (조립 자격)** — 조립체의 하우징에 일부러 물을 채우거나(또는 하우징 공기 배수량만큼 추를 더해) 물에 넣고, **72시간 똑바로 떠 있으면 합격.** VERIFIED_BUILD_TEST의 **T2(72h swamp)** 와 동일 조건으로 수행한다.

F1·F2 **둘 다 통과**해야 그 부력체는 **Verified Float**이며, 유닛의 `verified_build`에 반영된다.

## 3. 권장 자가 공수 소재 (업사이클 우선)

| 우선 | 소재 | 메모 |
|---|---|---|
| ◎ | **폐어구 부표 / 폐부표재** | 이미 부력 목적 설계, 회수-업사이클 서사와 시너지. ★사용이력품은 F1 필수(물먹음) |
| ◎ | **EVA 폼**(슬리퍼·매트·포장재), 수영 누들, 폐쇄셀 포장재 | 저렴·전세계·폐쇄셀 |
| ○ | **밀봉 PET 병** | 단기 캠페인. UV 열화·미세플라스틱 주의 |
| ○ | **밀폐 HDPE/PP 용기**(락앤락 카테고리) | 플라스틱 자체가 뜸(ρ<1)이라 침수 페널티 작음·재사용 |
| ✕ | 개방셀 폼·부스러지는 EPS 알갱이·미처리 목재·흡수성 소재 | F1 탈락 대상 |

> **환경 정직**: 자가 공수를 **업사이클(폐어구·폐포장재)** 로 유도하면 다양성이 곧 환경 플러스가 된다. 소재 불문 **회수-우선**([PRELAUNCH_RECOVERY_CHECKLIST.md](PRELAUNCH_RECOVERY_CHECKLIST.md))은 그대로 적용된다.

## 4. provenance 게이트 연동

부력체가 이 표준(F1·F2)을 통과한 유닛만 데이터로 인정된다 → "제대로 공수·검증할 유인". 게이트 로직은 [PRELAUNCH_RECOVERY_CHECKLIST.md](PRELAUNCH_RECOVERY_CHECKLIST.md) 파트 3 참조.
